# RULES.md - 操作规则

> **规则不会妥协。违反即错误。**

---

## 1. 熵减规则（核心）

### 1.1 文件创建约束
- **禁止**创建仅包含单次会话内容的新文件
- **优先**追加到现有主题文件
- **例外**：新项目的首个文档、系统文件

### 1.2 内容更新约束
- **必须**使用 `memory_update` 进行精准替换
- **禁止**全文覆盖式写入（除非文件 < 100 行）
- **必须**保留原有结构标记（如标题层级）

### 1.3 链接规则
- **必须**使用 `[显示文本](file:///绝对路径)` 格式
- **禁止**相对路径（如 `./file.md`）
- **必须**包含行号锚点（如 `#L12-L34`）用于精确引用

---

## 2. 命名规则

### 2.1 文件命名
| 类型 | 格式 | 示例 |
|------|------|------|
| 每日记录 | `YYYY-MM-DD.md` | `2026-04-18.md` |
| 项目文档 | `{project-name}-{type}.md` | `harness-design.md` |
| 索引文件 | `INDEX-{scope}.md` | `INDEX-projects.md` |
| 归档文件 | `ARCHIVE-{date}-{topic}.md` | `ARCHIVE-2026-04-chenghuiping.md` |

### 2.2 标题命名
- **一级标题** = 文件主题
- **二级标题** = 主要章节
- **三级标题** = 具体内容点
- **禁止**跳级（如一级直接到三级）

---

## 3. 索引规则

### 3.1 必须索引的内容
- 所有项目文档
- 关键决策记录
- 方法论沉淀
- 待办事项（未完成）

### 3.2 索引格式
```markdown
## {主题}

| 文件 | 关键内容 | 状态 |
|------|----------|------|
| [文件名](file:///path) | 一句话摘要 | 活跃/归档 |
```

### 3.3 索引更新时机
- 新文件创建时
- 文件状态变更时（活跃→归档）
- 每日心跳检查时

---

## 4. Skill 调用规则

### 4.1 触发条件
| Skill | 触发条件 |
|-------|----------|
| `/harness.summarize` | 会话结束、上下文达80%、用户指令 |
| `/harness.reflect` | 项目里程碑、用户指令、每周一次 |
| `/harness.retrieve` | 会话开始、遇到相关问题、用户指令 |
| `/harness.archive` | 项目结束、文件数>20、用户指令 |

### 4.2 禁止行为
- **禁止**在单次会话中多次调用同一 Skill
- **禁止**Skill 嵌套调用（Skill 调用 Skill）
- **禁止**无用户确认的执行归档操作

---

## 5. 日志规则

### 5.1 必须记录的事件
- 系统规则变更
- 索引结构变更
- 归档操作
- 错误和异常

### 5.2 日志格式
```markdown
## [YYYY-MM-DD HH:MM] {事件类型}

**触发**: {原因}
**操作**: {具体动作}
**结果**: {状态}
**影响**: {关联文件}
```

---

## 6. 合规检查清单

每次操作前自检：

- [ ] 是否违反了熵减规则？
- [ ] 命名是否符合规范？
- [ ] 索引是否需要更新？
- [ ] 是否需要记录日志？
- [ ] 链接格式是否正确？
