# SKILL.md - /harness.reflect

> **模式识别：从经验中提取规律。**

---

## 功能

触发自我反思，识别决策模式、验证假设、沉淀方法论。

反思类型：
- **事后复盘**：项目/阶段完成后
- **模式识别**：发现重复出现的决策场景
- **假设验证**：检验之前的预判是否准确
- **方法论沉淀**：提取可复用的工作方法

---

## 触发条件

1. 项目里程碑完成
2. 关键决策被验证（成功或失败）
3. 用户主动指令："/harness.reflect"
4. 每周自动触发一次（低优先级）

---

## 执行流程

```
1. 读取当前项目/主题的历史记录
2. 识别相似场景（时间、决策、结果）
3. 对比本次与历史的差异
4. 回答核心反思问题
5. 输出洞察和建议
6. 更新方法论索引（如有新发现）
```

---

## 核心反思问题

### 事后复盘
1. 预期目标是什么？实际结果如何？
2. 关键差异点在哪里？
3. 哪些决策被验证了？哪些需要修正？
4. 如果重来，会怎么做不同？

### 模式识别
1. 这个场景之前出现过吗？
2. 当时的决策和结果是什么？
3. 有什么规律可以提取？
4. 能否建立决策 checklist？

### 假设验证
1. 之前做了什么预判？
2. 验证结果如何？
3. 置信度需要调整吗？
4. 有什么新假设需要建立？

---

## 输出格式

```markdown
## 反思记录 [YYYY-MM-DD] - {类型}

### 背景
{触发反思的场景}

### 历史对比
| 时间 | 场景 | 决策 | 结果 |
|------|------|------|------|
| {日期} | {简述} | {选择} | {结果} |

### 洞察
1. {关键发现}
2. {模式识别}
3. {建议改进}

### 行动项
- [ ] {具体改进措施}

### 方法论更新
- {如有新方法论，更新 INDEX.md}
```

---

## 约束

1. **单次会话只触发一次** reflect
2. **必须**基于实际数据，不凭空推测
3. **必须**链接到具体记忆文件作为证据
4. **禁止**过度反思（如无实质洞察，记录"无显著模式"即可）
