# SKILL.md - /harness.retrieve

> **精准检索：找到需要的信息，不多不少。**

---

## 功能

基于当前上下文，智能检索相关历史记忆。

检索维度：
- **语义相似度**：内容相关度（核心）
- **时间邻近性**：近期记忆加权
- **项目关联性**：同一主题聚合
- **用户显式指定**：按项目/时间/类型过滤

---

## 触发条件

1. 会话开始时（自动检索相关背景）
2. 遇到需要历史上下文的问题
3. 用户主动指令："/harness.retrieve {查询}"
4. 生成摘要前（检索相关记忆用于关联）

---

## 执行流程

```
1. 提取当前上下文关键词
2. 调用 memory_search 进行语义检索
3. 按多维度排序（语义+时间+项目）
4. 过滤低相关性结果
5. 输出结构化检索结果
6. 可选：读取具体片段内容
```

---

## 检索语法

用户指令支持：

```
/harness.retrieve {关键词}              # 语义检索
/harness.retrieve project:{项目名}       # 按项目过滤
/harness.retrieve since:{日期}          # 时间范围
/harness.retrieve type:{decision|todo}  # 按类型过滤
/harness.retrieve {关键词} limit:{N}    # 限制结果数
```

---

## 输出格式

```markdown
## 检索结果 [YYYY-MM-DD HH:MM]

**查询**: {用户查询}
**匹配数**: {N} 条

### 高相关
1. [{文件名}](file:///绝对路径#L{行号})
   - 摘要: {一句话内容}
   - 相关度: {score}
   - 时间: {日期}

### 中相关
...

### 建议
{基于检索结果的下一步建议}
```

---

## 排序算法

```
最终得分 = 语义相似度 * 0.6 + 时间衰减 * 0.3 + 项目匹配 * 0.1

时间衰减 = 1 / (1 + ln(天数+1))
项目匹配 = 1（同项目）或 0（不同项目）
```

---

## 约束

1. **默认返回最多 5 条** 高相关结果
2. **必须**包含文件路径和行号锚点
3. **必须**提供一句话摘要，不展示全文
4. **禁止**返回低相关结果（score < 0.5）
