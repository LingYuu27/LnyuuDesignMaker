# SKILL.md - /harness.summarize

> **会话熵减：提取关键，丢弃噪声。**

---

## 功能

将会话内容压缩为结构化摘要，遵循熵减原则：
- 提取关键决策和待办
- 丢弃闲聊和中间过程
- 链接到已有记忆，不重复存储

---

## 触发条件

1. 会话自然结束（用户说"先这样"、"结束"等）
2. 上下文达到 80% 容量
3. 用户主动指令："/harness.summarize"
4. 项目阶段完成时

---

## 执行流程

```
1. 读取会话历史
2. 识别关键决策点（为什么做 > 做了什么）
3. 识别待办事项（完成状态）
4. 检索相关历史记忆
5. 生成结构化摘要
6. 写入指定位置（优先追加，不新建文件）
```

---

## 输出格式

```markdown
## 会话摘要 [YYYY-MM-DD HH:MM]

### 关键决策
| 决策点 | 选择 | 原因 |
|--------|------|------|
| {问题} | {答案} | {简要原因} |

### 待办状态
- [ ] {未完成项} → 迁移到 [INDEX.md](../../INDEX.md)
- [x] {已完成项} → 归档

### 关联记忆
- 引用: [{文件名}](file:///绝对路径#L{行号}) - {关键信息}

### 备注
{需要额外说明的内容}
```

---

## 约束

1. **禁止**创建仅包含本摘要的新文件
2. **必须**追加到现有项目文件或 `memory/YYYY-MM-DD.md`
3. **必须**更新 [INDEX.md](../../INDEX.md) 中的待办状态
4. **必须**使用 `[显示文本](file:///绝对路径)` 格式链接

---

## 示例

用户输入：
> "我们决定用 React 而不是 Vue，因为团队熟悉度更高。记得下次把组件库选型也定下来。"

输出：
```markdown
## 会话摘要 [2026-04-18 15:30]

### 关键决策
| 决策点 | 选择 | 原因 |
|--------|------|------|
| 前端框架选型 | React | 团队熟悉度更高 |

### 待办状态
- [ ] 组件库选型 → 迁移到 INDEX.md

### 关联记忆
- 无
```
