## 一、记忆层级架构

| 层级 | 内容 | 存储位置 | 更新频率 | 保留策略 |
|------|------|----------|----------|----------|
| **L1 原始对话** | 完整消息流 | 会话上下文 | 实时 | 会话结束即丢弃 |
| **L2 当日摘要** | 逐条意图+关键决策 | `memory/YYYY-MM-DD-{topic}.md` | 每次保存时 | 保留30天 |
| **L3 长期记忆** | 用户档案、工作模式 | `MEMORY.md` | 重要变更时 | 永久保留 |
| **L4 技能知识** | 通用教训、可复用方法 | `AGENTS.md`, `SKILL.md` | 经验固化时 | 永久保留 |

---

## 二、关键原则

### 2.1 意图优先
- **表面意图**: 用户明确说的内容
- **深层意图**: 用户为什么这么说（需求/痛点/目标）
- **记录方式**: 意图拆解 → 核心意图总结

### 2.2 逐条记录
- ❌ 不要: "用户要求保存对话"
- ✅ 要: "第N条 - 用户: '把所有对话流压成摘要' → 意图: 上下文即将耗尽，需要完整保存状态"

### 2.3 版本追踪
- 保存错误→修正的完整路径
- 标注每次迭代的变化原因
- 保留决策演进过程

### 2.4 数据完整
- "不要丢数据" = 所有决策、反馈、修正必须保留
- 用户纠正 → 必须记录纠正内容和原因
- 我的错误 → 必须记录错误类型和教训

## 三、自主执行流程

### 3.1 触发检测（我主动检测）

**必须保存的信号**:
- [ ] 用户说: "换窗口"/"上下文极限"/"保存"/"固化"
- [ ] 对话轮次 > 15
- [ ] 用户明确纠正我的错误
- [ ] 达成关键决策或结论
- [ ] 收到长文本输入 (>3000字符)
- [ ] 用户要求整理/归纳/总结

**建议保存的信号**:
- [ ] 对话持续 > 30分钟
- [ ] 涉及多个主题切换
- [ ] 有重要参考资料被引用

### 3.2 保存执行（我主动执行）

```
检测到保存信号
    ↓
[Step 1] 读取现有记忆文件
    - 检查 memory/YYYY-MM-DD-{topic}.md 是否存在
    - 检查 MEMORY.md 是否需要更新
    ↓
[Step 2] 提取关键信息
    - 逐条对话意图拆解 (L2)
    - 关键决策点提取 (L2/L3)
    - 用户偏好/模式观察 (L3)
    - 我的错误/教训 (L4)
    ↓
[Step 3] 写入文件
    - 追加到当日摘要文件
    - 更新长期记忆 (如需要)
    - 更新技能知识 (如需要)
    ↓
[Step 4] 确认报告
    - 报告保存的文件路径
    - 摘要保存的核心内容
    - 提示下次恢复位置
```

### 3.3 质量自检（保存后自检）

- [ ] 是否包含所有用户消息?
- [ ] 是否拆解了每条意图?
- [ ] 关键决策是否突出?
- [ ] 我的错误是否记录?
- [ ] 文件路径是否正确?
- [ ] 是否可追溯到原始内容?

## 六、长期记忆更新规则

### 何时更新 MEMORY.md
- 用户偏好变更（如记忆粒度要求）
- 工作模式调整（如沟通风格）
- 重要项目信息（如广州设计项目）
- 关键约束（如"不要丢数据"）

### 何时更新 AGENTS.md / SKILL.md
- 发现可复用的工作方法
- 总结错误教训形成规范
- 提炼通用设计原则
