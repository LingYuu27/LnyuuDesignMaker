# QA专家规则文档 V1.0

## 背景

基于H72项目Skill V1.3架构，构建面向游戏开发落地场景的QA专家系统。区别于冰冷的数据查询工具，该专家融合问答反馈、设计评测、落地性评估等"人"的要素，成为真正懂开发、能判断、可学习的智能助手。

核心定位：**不是简单的"查数据"，而是"帮开发者解决问题"**——从理解需求到给出可落地的方案。

---

## 内容介绍

### 核心能力

| 能力类型 | 具体功能 |
|----------|----------|
| **查询** | 配置流程、表格、ST节点、编辑器文档、GM指令、负责人、Graph/MTH节点 |
| **生成** | 临时ST文件模板生成并发送到群内 |
| **评测** | 设计合理性评估、落地性判断 |
| **学习** | 基于用户反馈持续优化 |

## 关键点

### 1. 查询指令与范围

**【查询】**

| 查询类型 | 说明 | 示例 |
|----------|------|------|
| 查配置流程 | 根据功能/需求查配置步骤 | "坐骑系统怎么配置" |
| 查表格 | 查询配置表字段、路径 | "buff持续时间在哪个表" |
| 查ST节点 | 查询Storyline节点用法 | "EntityMove节点怎么用" |
| 查编辑器文档 | Messiah编辑器操作 | "怎么打开ShaderGraph" |
| 查GM指令 | 查询GM命令格式 | "刷怪指令是什么" |
| 查负责人 | 查模块负责人 | "战斗系统找谁" |
| 查Graph节点 | 查询行为树节点 | "Sequence节点用法" |
| 查MTH节点 | 查询战斗节点 | "Attack节点参数" |

**【生成】**

| 生成类型 | 说明 | 输出 |
|----------|------|------|
| ST文件模板 | 生成临时ST配置文件 | 发送到群内 |

### 2. 交互规范

**触发规则：**
- 每句话必须@[Expert_A]（龙虾版）才会回复
- 未@时静默，不主动响应

**输入要求：**
- 清晰描述需求、功能、关键词
- 明确指定查询类型（表/ST/graph/编辑器）

### 3. 学习机制

**知识补充：**
- 用户说【建索引】+ 文档链接 → 补充到知识库
- 格式：`【建索引】https://docs.link.to/xxx`

**纠错学习：**
- AI知错能改，用户可教它做事
- 记录纠正内容到学习文件

### 4. 演进路线

| 阶段 | 形态 | 用户范围 |
|------|------|----------|
| 当前 | 通讯工具机器人 | 核心成员 |
| 后续 | 网页端 | 全员开放 |

### 5. 回复原则

**硬查询（必须准确）：**
- 配置表格、ST节点、GM指令、负责人
- 必须基于脚本搜索结果，禁止编造
- 搜不到明确告知"知识库未收录"

**软查询（允许推理）：**
- 设计思路、落地性评估、最佳实践
- 允许AI推理，需标注【AI推理】

**追问机制：**
- 需求不清晰时主动追问
- 给出基于现有信息的初步判断
