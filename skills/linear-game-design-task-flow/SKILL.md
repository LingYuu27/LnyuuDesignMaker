# 线性流程游戏设计：拆环与落地规范

> 从文案台本到任务表配置的完整流程规范

---

## 一、拆环核心原则

### 1.1 最小单元原则

**定义**：拆环必须拆分到**最小单元——即一环**，拆环文档的环数与最终任务表里的环数尽量一致。

**判断标准**：
| 场景 | 判断依据 | 环数 |
|-----|---------|------|
| 交互+对话 | 交互本身是否有叙事含义 | 2环（有意义）/ 1环（纯触发器） |
| 连续对话 | 能否找到合适的玩家行为插入 | 能→拆分 / 不能→合并 |
| 调查+反馈 | 调查动作本身是否有表现 | 有→单独环 / 无→合并 |

**反例**：
- ❌ 交互只是开启对话的触发器，却拆成2环
- ❌ 连续3段对话无玩家行为，却拆成3环
- ❌ 拆环文档10环，任务表实际20环（设计阶段没想清）

---

### 1.2 玩家行为节奏原则

**定义**：避免连续重复的体验（对话-对话-对话，交互-交互-交互）。

**节奏模式**：
```
对话 → 交互 → 对话 → 移动 → 对话 → 战斗 → 对话
（交替进行，保持玩家参与感）
```

**处理方法**：
| 问题 | 解决方案 | 示例 |
|-----|---------|------|
| 连续对话 | 插入重新交互/提交道具 | "和NPC重新交互发起对话" |
| 连续交互 | 合并为1环或插入对话 | 调查3个物品→合并或每调查1个对话反馈 |
| 长对话 | 拆分为多环，中间插入行为 | 对话A→让玩家移动→对话B |

---

### 1.3 信息顺序原则

**定义**：玩家获得信息的顺序应与游戏内主角保持统一。

**标准流程**：
```
剧情交代完成 → 主角行为动机转变 → 任务面板刷新
```

**反例修正**：
- ❌ "那边好像有东西，过去看看"放在下一环任务接取后
  - 问题：玩家先看到面板刷新，再看到对话
  - ✅ 修正：合并到上一段对话结尾，对话播完再刷新任务

- ❌ NPC说"快回去吧，小心又要罚跪"，然后任务才刷新
  - 问题：玩家动机已转变，但任务未刷新
  - ✅ 修正：cue主角行为动机先转变，任务再刷新

## 二、对话表现标准选择

### 2.1 表现层级

| 层级 | 标准 | 适用场景 | 资源成本 |
|-----|------|---------|---------|
| L1-无镜头对话 | 22号标准，动态焦点镜头 | 默认标准，铺量内容 | 低 |
| L2-关键帧插入 | 无镜头对话中插入1-2段关键镜头 | 必须使用镜头的情节 | 中 |
| L3-黑屏文字 | 黑屏+文字描述过程 | 过程性内容，保底方案 | 低 |
| L4-镜头对话 | 完整镜头演出，需QD审核 | 重要情节，高制作标准 | 高 |

### 2.2 选择决策树

```
台本表现需求
    ↓
能否简化不影响剧情？
    ↓是→ 和文案沟通简化
    ↓否
能否用关键帧插入解决？
    ↓是→ L2关键帧插入（首选）
    ↓否
是否频繁使用黑屏？
    ↓否→ L3黑屏文字（保底）
    ↓是
是否特别重要且有语音？
    ↓是→ L4镜头对话（提档审核）
    ↓否→ 回到L2或L3
```

### 2.3 关键帧使用规范

**适用场景**：
- NPC从树上跳下来（入场表现）
- 玩家使用特定道具
- 场景切换过渡
- 情绪高潮点

**使用方式**：
- 在无镜头对话的镜头轨道上叠加关键帧
- 关键帧结束后回到动态焦点镜头（标准无镜头对话）
- 控制数量：1-2段/环，避免频繁打断

## 三、任务点位布置规范

### 3.1 动线逻辑

**原则**：尽量避免连续任务在一两个点之间反复来回。

**检查方法**：
```
绘制任务点位图 → 标记移动路径 → 识别反复来回 → 优化点位分布
```

**优化策略**：
| 问题 | 解决方案 |
|-----|---------|
| A→B→A→B | 合并为A→B→C，或调整任务顺序 |
| 长距离往返 | 中间插入中间点，或改为并行任务 |
| 单点聚集 | 分散到不同子区域，利用场景变化 |

### 3.2 点位选择标准

**禁止**：
- ❌ NPC站在草里
- ❌ NPC站在水里
- ❌ NPC站在奇怪角落
- ❌ NPC站在不平坦斜坡

**要求**：
- ✅ 选择玩家靠近后表现可控的点位
- ✅ 配置符合环境氛围的KIT情境
- ✅ 考虑玩家可能从任意方向靠近

### 3.3 KIT情境配置

**原则**：在任何玩家可以操作的窗口期，NPC都要有真实可信的行为表演。

**标准KIT类型**：
| 情境 | 行为 | 示例 |
|-----|------|------|
| 等待 | 原地等待KIT | NPC让玩家去找东西，原地等待 |
| 工作 | 特定动作KIT | 红线抄书，真的在抄书 |
| 移动 | 路径巡逻KIT | NPC在区域内来回走动 |
| 交互 | 与环境互动KIT | NPC在摊位前挑选商品 |

**反例**：
- ❌ NPC让玩家去找东西，却干站在原地
- ❌ 对话说"我在忙"，但KIT是待机
- ❌ 夜间任务，NPC白天KIT未切换

**目的**：检查连续任务体验是否存在重复

**标记规则**：
| 色块 | 类型 | 检查重点 |
|-----|------|---------|
| 🟦 蓝色 | 对话环 | 避免连续3个蓝色 |
| 🟩 绿色 | 交互环 | 避免连续2个绿色 |

## 六、常见问题处理

### Q1：台本写了一些不好落地的表现，必须借助镜头，怎么办？

**优先级处理**：
1. **优先沟通**：和文案协商，简化不影响剧情传达的表演
2. **首选方案**：在无镜头对话中插入一小段关键帧镜头
3. **保底方案**：用黑屏+文字描述过程
4. **提档方案**：特别重要的情节提档为镜头对话（需QD审核）

### Q2：遇到连续的对话怎么处理？

**判断**：
- 如果必须连在一起 → 合并为一段
- 如果能找到玩家行为 → 拆分，插入交互/移动/提交道具

### Q3：交互+对话，怎么判断是1环还是2环？

**判断标准**：
- 交互本身有叙事含义/有表现 → 2环
- 交互只是开启对话的触发器 → 1环（用[给NPC附加通用交互]结构）

### Q4：任务点位怎么选择？

**标准**：
- 避免草里、水里、奇怪角落、不平坦斜坡
- 选择玩家靠近后表现可控的点位
- 配置符合环境氛围的KIT
- 考虑动线逻辑，避免反复来回

## 七、与多模态评测的衔接

### 7.1 在评测流程中的位置

```
概念设计（多模态评测）
    ↓
拆环设计（本规范）
    ↓
原型验证（玩法测试）
    ↓
数据反馈（回流优化）
```

### 7.2 对多模态评分的影响

| 多模态维度 | 拆环落地影响 | 调整权重 |
|-----------|-------------|---------|
| 视觉模态 | 对话表现标准选择 | ±10% |
| 玩法模态 | 玩家行为节奏设计 | ±15% |
| 叙事模态 | 信息顺序与节奏 | ±10% |
| 技术模态 | 表现标准资源成本 | ±10% |
| 商业模态 | 无直接影响 | - |

**示例**：
```
概念评分：玩法模态 8/10（设计有趣）
拆环检查：连续3段对话无玩家行为
调整后：玩法模态 6/10（节奏问题）
```

## 八、总结

拆环与落地规范的核心是：**从概念到配置的可执行性**。

它不是来评判"设计好不好"，而是确保：
- 设计能拆分到**可配置的最小单元**
- 表现标准选择**符合资源约束**
- 玩家体验节奏**避免疲劳**
- NPC行为**全程可信**
- 信息传递**符合逻辑**

**关键原则**：
1. 最小单元：拆到一环，文档与任务表一致
2. 节奏交替：避免连续重复体验
3. 信息顺序：玩家=主角，动机先于任务刷新
4. 表现务实：L1默认，L2首选，L3保底，L4审核
5. 点位合理：可控表现，真实KIT，优化动线
