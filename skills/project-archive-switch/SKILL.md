# 项目归档与切换 SKILL

> 版本: v1.0
> 用途: 标准化项目失败后的归档流程和上下文切换

---

## 触发条件

- 项目终止（无论内因/外因）
- 用户指令"整理记忆系统"
- 用户预告"开启新项目"

---

## 执行流程

### Step 1: 失败复盘（5分钟）

```
用户告知失败
    ↓
[1] 确认失败阶段（初试/复试/终面）
[2] 询问具体反馈（结构/归纳/TD/其他）
[3] 提取可复用资产（技术方案、数据表）
[4] 记录教训（具体到可执行改进点）
```

**输出**: `{日期}-project-archive.md`

---

### Step 2: 资产提取（10分钟）

**筛选标准**:
| 保留 | 剔除 |
|------|------|
| 引擎组件映射 | 文案描述 |
| 数据表结构 | 文化考据 |
| 状态机设计 | 背景故事 |
| 性能预算 | 情感渲染 |

**输出**: `{项目名}技术方案.md`

---

### Step 3: 记忆系统整理（5分钟）

```
读取 memory/ 文件夹
    ↓
按主题分类统计
    ↓
输出四层架构现状
    ↓
标记待归档/待清理/待保留
```

**输出**: 内存中的结构化总结

---

### Step 4: 新项目准备（2分钟）

```
创建新项目目录
    ↓
写入README占位
    ↓
更新MEMORY.md长期记忆
```

---

## 文件清单模板

| 类别 | 文件 | 大小 | 处置 |
|------|------|------|------|
| 系统协议 | *.md | - | 保留 |
| 项目归档 | *-archive.md | - | 保留 |
| 技术资产 | *技术方案.md | - | 保留 |
| 过程文档 | 草稿、中间版本 | - | 评估后清理 |

---

## 质量检查

- [ ] 失败原因记录具体（非"结构不好"而是"亮点未前置"）
- [ ] 技术资产可复用（非一次性）
- [ ] 记忆系统总结完整（四层架构全覆盖）
- [ ] 新项目目录已创建
- [ ] MEMORY.md已更新

---

## 教训库

| 场景 | 错误 | 改进 |
|------|------|------|
| 初试失败 | 亮点后置 | 30秒法则：核心结论前置 |
| TD不足 | 只有概念 | 补充引擎组件、性能预算、验收用例 |
| 归纳不足 | 平铺描述 | 显性化分类矩阵 |
